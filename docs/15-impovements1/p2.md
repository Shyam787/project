# Source Visibility Improvements

## Problem

Current citations show:

```text
[c1]
[c2]
```

This does not immediately tell users where information came from.

Enterprise users want to know exactly which document produced an answer.

---

## Requirement

Every citation must display source information.

Current:

```text
[c1]
```

Required:

```text
[c1] AccessPolicy.pdf
```

or

```text
[c1] RemoteWorkPolicy.docx
```

---

## Citation Metadata

When users expand a citation, display:

```text
Document Name
Document Classification
Uploader
Upload Date
```

Example:

```text
AccessPolicy.pdf
Classification: Internal
Uploaded By: admin@acme.com
Uploaded On: 2026-06-01
```

---

## Source Preview

Users must be able to preview the cited chunk directly.

Preview panel should show:

```text
Document Name
Chunk Content
Classification
```

Users should not need to navigate away from the Ask page.

# Better No-Evidence Experience

## Problem

Current behavior may return a generic message when no evidence is found.

This creates uncertainty.

Users do not know whether:

* data does not exist
* permissions blocked access
* retrieval failed
* question is unclear

---

## Required Response

When no grounded answer exists:

Display:

```text
No authorized information was found that answers this question.
```

Followed by:

```text
Possible reasons:

• The information does not exist in uploaded documents
• You may not have permission to access the relevant document
• The question may need to be rephrased
• The document may not have been uploaded yet
```

---

## Hallucination Prevention

The system must never generate invented answers when evidence is unavailable.

Required behavior:

```text
No evidence
→ No answer generation
→ Explain why
```

Never fabricate information.

# Document Status Visibility

## Purpose

Administrators need immediate visibility into document state.

---

## Status Types

Every document must display one of:

```text
Processing
Indexed
Failed
Archived
Soft Deleted
```

---

## Status Display

Status must be visible:

* document table
* document details page
* document preview

Use clear visual indicators.

---

## Processing State

While indexing:

Display:

```text
Processing
```

and prevent querying against incomplete indexing jobs.

---

## Failed State

If extraction or indexing fails:

Display:

```text
Failed
```

and provide error details for administrators.

# Out Of Scope

The following must not be implemented in P2:

* billing systems
* payment gateways
* tenant branding engines
* workflow approval systems
* custom role builders
* analytics platforms
* business intelligence dashboards
* email campaign systems
* AI-generated document summaries
* advanced reporting systems

These features are intentionally excluded to keep the platform focused, maintainable, and aligned with its core purpose.

The purpose of this platform remains:

A secure, multi-tenant, role-governed Enterprise RAG system that provides grounded answers from authorized enterprise knowledge sources.
